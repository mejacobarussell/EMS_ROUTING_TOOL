# EMSTOOL | Clinical Operations & Tactical Mapping

**EMSTOOL** is a high-performance, high-contrast mission control interface designed for EMS field providers and agency administrators. It centralizes critical hospital data, station mapping, and intelligent routing into a single, reliable platform.

> [!IMPORTANT]
> **Access Model Update:** EMSTOOL is now provided exclusively as a managed service. This repository serves as the official documentation and landing page. The source code is no longer public to ensure the stability and integrity of our global routing engines and clinical datasets.

---

## 🚑 Key Features

### 1. Intelligent Hospital Intelligence
Never guess which ER is best for your patient. EMSTOOL provides real-time data at a glance:
* **Live Routing:** Powered by the **OSRM-DFW** engine for precise arrival estimates.
* **Clinical Capabilities:** Instant filtering for **STEMI, Stroke (PSC), Peds, and Trauma** centers.
* **Field Intelligence:** Shared "EMS Room" notes—know which hospitals have the best snacks and seating before you arrive.

### 2. Post & Station Management
Customized workspaces (such as our **fortworth** deployment) allow agencies to organize resources by:
* **Battalion & Station:** Grouping units and posts for localized situational awareness.
* **One-Touch Navigation:** "NAV" buttons bridge the gap between the tool and your device's native mapping for immediate response.

### 3. Mission-Ready Interface (NOC Aesthetic)
Designed for high-intensity environments where eye strain and speed matter:
* **High-Contrast UI:** Optimized for visibility in both direct sunlight and dark ambulance cabs.
* **DIM Mode:** A dedicated toggle to maintain "stealth" in the cab during night shifts.
* **System Stability Bars:** Real-time monitoring of CPU, Physical Memory, and Swap ensures the tool remains responsive when seconds count.
